## What this repo is

Clean-sheet reimplementation of Apache Subversion in Aether. Not a
mechanical CŌåÆAether translation ŌĆö the reference svn C tree lives at
`subversion/libsvn_*` as a *behaviour contract*, read-only. We
wrote our own on-disk formats, wire protocol, algorithms.

Three production binaries: `svn` (client CLI), `aether-svnserver`
(HTTP server), `svnadmin` (repo admin). Several test binaries.

## Layout

- `ae/<area>/*.ae` ŌĆö first-party Aether source, compiled with
  `aetherc --emit=lib` and linked into the C shims.
- `ae/<area>/*_shim.c`, `ae/<area>/shim.c` ŌĆö hand-written C that
  owns FFI glue, opaque handles, and I/O where Aether can't reach.
- `ae/<area>/*_generated.c` ŌĆö aetherc output. **Gitignored**.
  Regenerated by `./regen.sh` from the paired `.ae`.
- `ae/<area>/Makefile.regen` ŌĆö per-area recipe; the top-level
  `regen.sh` runs every one of them.
- `aether.toml` ŌĆö bin manifest. Each `[[bin]]` lists extra_sources
  (the C shims + _generated.c files it links). Keep it literal ŌĆö
  adding a new `.ae` means adding its `_generated.c` to every bin
  that uses the matching shim.
- `PORT_STATUS.md` ŌĆö progress narrative, updated at the end of each
  round. Read the top 60 lines at session start.
- `AETHER_ISSUES.md` ŌĆö running list of Aether-side bugs / workarounds.
  Prune as issues resolve upstream. Stays terse.

## The port's methodology

**MDOLD: Most-Depended-On, Least-Depending.** Leaf-first CŌåÆAether.
Each commit is one function or one small cluster. Tests must stay
green. "Baby commits" ŌĆö typically 20-80 lines of C out, 20-50 lines
of `.ae` in, one Aether wrapper added if needed.

Current C/Aether ratio tracked by:
```
c=$(git ls-files 'ae/**/*.c' 'ae/**/*.h' | xargs wc -l | tail -1 | awk '{print $1}')
ae=$(git ls-files 'ae/**/*.ae' | xargs wc -l | tail -1 | awk '{print $1}')
python3 -c "print(f'C share: {$c*100/($c+$ae):.2f}%')"
```
Generated `.c` files are gitignored, so this is GitHub-stats accurate.

## Aether specifics you need (in priority order)

1. **`--emit=lib`** produces FFI-ABI C. All our `.ae` files use it.
2. **`--with=fs` / `--with=net` / `--with=os`** ŌĆö capability opt-ins.
   `std.fs` etc. are banned in `--emit=lib` by default. We are the
   host ŌĆö always pass `--with=fs` in `Makefile.regen` for any `.ae`
   that uses `std.fs`, `std.dir`, or `std.file`. `ae/subr/
   Makefile.regen` is the pattern.
3. **Go-style returns** ŌĆö `err = fs.write_atomic(...)`; empty string
   = success.
4. **Binary-safe reads** go through `fs_try_read_binary` +
   `fs_get_read_binary` + `fs_get_read_binary_length` +
   `fs_release_read_binary` (TLS-buffered), **not** `fs.read_binary`
   which concats through a C-string path and truncates at NULs.
   `ae/fs_fs/rep_store_shim.c` and `ae/wc/pristine_shim.c` use the
   raw API for this reason.
5. **Reserved keywords**: `state`, `match`, `message`. Rename to
   `st`, `is_match`, `msg`. Fails silently-ish at parse time; the
   extern-signature call that errors will be the hint.
6. **`export` fn can't call another `export` fn in the same file.**
   Split into `foo_impl()` + `export foo() { return foo_impl() }`.

## The shared Aether io layer

`ae/subr/io.ae` is the port's atomic-write / mkdir-p / slurp / stat
/ listdir wrapper over `std.fs`. Everything that used to have a
hand-rolled `write_file_atomic` / `mkdir_p` / `slurp_small` now
calls through:

- `aether_io_write_atomic(path, data, len) -> 0/1`
- `aether_io_mkdir_p(path) -> 0/1`
- `aether_io_read_file(path) -> string` (NUL-terminated; caller
  checks `io_file_size` separately to distinguish missing from empty)
- `aether_io_file_size(path) -> int` (-1 if missing)
- `aether_io_is_regular_file(path)`, `_is_directory`, `_exists`
- `aether_io_stat_kind(path) -> int` (1=file 2=dir 3=symlink 4=other
  0=missing)
- `aether_io_listdir(path) -> void*` + `_count(h)` + `_name(h, i)` +
  `_free(h)`

Link `ae/subr/io_generated.c` into any bin whose shim uses these.
Pattern established; check `aether.toml` ŌĆö most binaries have it
already.

## Build & test

- `./regen.sh` ŌĆö regenerate every `_generated.c`. Invoked by
  `./build.sh` automatically.
- `./build.sh` ŌĆö regen + `ae build` every `[[bin]]`. Output binaries
  go to `/tmp/<name>`.
- `./build.sh <name>` ŌĆö build one bin (e.g. `./build.sh svnadmin`).
- Shell tests: `ae/<area>/test_*.sh`. Each spins up what it needs
  (server for svnserver tests, fresh repo for commits, etc.). Run
  individually or in a loop.
- `ae/subr/test_*.ae` + `ae/wc/test_*.ae` + `ae/fs_fs/test_*.ae`
  + `ae/repos/test_*.ae` ŌĆö in-language tests, built as their own
  bins and run by the shell harnesses too.

Port conflicts: the svnserver tests use port 9300 by default. If
a previous run left a zombie, `pkill aether-svnserver` and retry.
Not a code bug; re-run is safe.

## The recursive-walker port pattern

For anyone porting another structural walker, the recipe that
worked for rebuild_dir / filter_dir / redact_dir / paths_changed /
update-apply / merge-apply / resolve_path:

1. Keep rep-store I/O + opaque handles in C. `svnae_rep_read_blob`,
   `svnae_rep_write_blob`, sqlite3 handles all stay C externs.
2. Represent any in-memory accumulator (reclist, flat_tree) as a
   newline-separated string in the `K SHA NAME\n` dir-blob format.
   Aether already has parsers for that format (`aether_dir_*`).
3. Add C-side `*_count / *_path_at / *_kind_at / *_sha_at / *_data_at /
   *_data_len_at / *_find_by_path` accessors to any list struct the
   Aether side needs to iterate. Mirror update_shim's `rtree_*` for
   the shape.
4. Two out-params ŌåÆ two exports (e.g. `resolve_kind` + `resolve_sha`).
   Walk the path twice; it's cheap.
5. Recursive functions use the `foo_impl()` + `export foo() { return
   foo_impl() }` split. Aether's export-can't-call-export rule is
   still there.
6. Reserved keywords: `state`, `match`, `message`, `ptr`, `else` as
   an identifier, `address`. `else` as part of `if/else` works
   (don't shadow `else`). When in doubt, use separate `if` checks.
7. After porting, add the `*_generated.c` to every `[[bin]]` in
   `aether.toml` that links the shim ŌĆö typically at least two or
   three bins.

## Don't

- Don't "improve" the Go-style `(value, err)` convention. Match it.
- Don't add `// removed foo` comments after deleting code. Delete
  cleanly; git log is the history.
- Don't batch ports into mega-commits. One leaf per commit, tests
  green between each.
- Don't let `aether.toml` drift from what regen produces. New
  `.ae` ŌåÆ new Makefile.regen entry ŌåÆ new `_generated.c` in every
  consuming bin.
- Don't skip the tests. `pristine`, `txn`, `repos`, and the
  `svnserver`/`svnadmin` shell tests are the integration gates.
